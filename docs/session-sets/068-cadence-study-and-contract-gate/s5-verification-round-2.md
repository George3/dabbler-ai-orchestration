ISSUES_FOUND

**Major 1 — Invalid UTF-8 still escapes the shared “never raises / unreadable” guard**  
- **Issue** → `_read_activity_entries()` catches `OSError` and `json.JSONDecodeError`, but not `UnicodeDecodeError`. A corrupt `activity-log.json` with invalid UTF-8 bytes will still raise out of `read_contract_gate()`, `has_contract_gate_record()`, and `contract_gate_record_unreadable()`. In the close-session fail-open wrapper, that can still suppress both the warning and the gate enforcement path.  
- **Location** → `ai_router/contract_gate.py:_read_activity_entries`  
- **Fix** → Catch `UnicodeDecodeError` (or `UnicodeError`) alongside `OSError` / `json.JSONDecodeError` and return `([], True)`. Add a test that writes invalid UTF-8 bytes and asserts: `read_contract_gate()` returns `DEFAULT_CONTRACT_GATE`, `has_contract_gate_record()` returns `False`, and `contract_gate_record_unreadable()` returns `True`, all without raising.

**Major 2 — Non-dict items inside `entries` still slip through as a clean “no record”**  
- **Issue** → A log like `{"entries": ["bad"]}` or `{"entries": [42]}` is structurally corrupt for this durable-record format, but `_read_activity_entries()` silently drops those items and reports `malformed=False`. That means `read_contract_gate()` returns `none`, `has_contract_gate_record()` returns `False`, and `contract_gate_record_unreadable()` returns `False` — so a corrupted durable record can still disappear with no warning.  
- **Location** → `ai_router/contract_gate.py:_read_activity_entries` and, transitively, `read_contract_gate()`, `has_contract_gate_record()`, `contract_gate_record_unreadable()`, `record_contract_gate()`  
- **Fix** → Treat any non-dict element in `entries` as corruption for unreadable detection. Keep the readers tolerant by filtering dicts for scanning, but surface a corruption flag when any non-dict is present; make `contract_gate_record_unreadable()` return `True` on that flag, and make `record_contract_gate()` reject logs whose `entries` list contains non-dict items. Add tests for `{"entries":["bad"]}` and mixed lists.
