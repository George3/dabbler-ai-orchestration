## Findings
- **High / Budget enforcement / `ai_router/pull_verifier.py:1337-1350`** / **Issue:** `_FORCE_VERDICT_BUDGET_FRACTION = 0.85` does not actually guarantee headroom for one more forced call. `near_budget` is only sampled between turns, so an unforced probe can jump from `<85%` to arbitrarily close to `100%` (for example, `84% -> 99%`). The next forced-verdict request may then exceed the remaining budget/cost, and because caps are only checked at loop top, a `submit_verdict` on that request will still terminate as `STOP_VERDICT` rather than a ceiling stop. This fails verification point (3) and weakens cap semantics on the near-budget path. **Fix:** trigger forcing based on explicit reserved headroom (`spent >= ceiling - reserve`) where `reserve` is a hard upper bound/conservative estimate for one forced request, or bound the forced request itself and re-check token/cost ceilings immediately after accounting that call before accepting `submit_verdict`.
